# コサイン類似度（Cosine Similarity）

> **一言でいうと**：「2つのベクトルの向きがどれだけ近いかを -1〜1 の数値で表す指標。1に近いほど意味が似ている」

---

テキストのクラスタリングを勉強していて、「文章の近さをどう測るか」という問題にぶつかりました。

最初は単純に文字の一致率を考えていたのですが、実際に試してみると「あれ、これ全然ダメだ」という場面に何度も遭遇しました。この記事では、そのつまずきをそのまま書きながら、コサイン類似度の仕組みと使いどころを整理しています。

---

## 最初のつまずき：「文字が似ている」≠「意味が似ている」

テキストの類似度というと、まず思い浮かぶのは文字の一致率です。同じ文字が多ければ似ている、というイメージ。でも、こんな例を考えてみてください。

```
「武器輸出に反対する」
「武器輸出に賛成する」
```

「反対」と「賛成」は1文字違いです。文字の編集距離（Levenshtein距離）で測ると、2つの文章は**かなり近い**と判定されます。でも意味は真逆ですよね。

逆のケースもあります。

```
「武器輸出に反対する」
「軍拡をやめろ」
```

共通する文字はほとんどない。文字の類似度で測ると**遠い**と判定されますが、どちらも反戦・平和を訴える文章です。意味としては近い。

この2つの例で「文字の類似度では意味を捉えられない」ことが直感的にわかります。では何を使えばいいのか、というのがコサイン類似度（とその前提となるEmbeddingベクトル）の話につながります。

---

## コサイン類似度とは何か

テキストをEmbeddingベクトルに変換すると、文章が数値の配列（ベクトル）として表現されます。意味が似た文章は、この高次元空間上で近い位置に集まります。

コサイン類似度は、**2つのベクトルの「向き」の近さ**を測る指標です。

```
cosine_similarity(A, B) = (A · B) / (|A| × |B|)
```

- **A · B**：内積（対応する要素を掛けて全部足す）
- **|A|**：ベクトルAの長さ（L2ノルム）
- **|B|**：ベクトルBの長さ（L2ノルム）

### 角度で考えると直感的

コサイン類似度は、2つのベクトルがなす角度 θ の cos(θ) に等しくなります。

```
         B         B         B              B               B
          ↑        ↗          ↗（同じ）       ↘             ↓
     θ=90°|   θ=45°      θ=0°          θ=135°        θ=180°
          |  /
    ───── A →──────────────────────────── A → ─────── A →

cos(90°)=0.0  cos(45°)=0.71  cos(0°)=1.0  cos(135°)=-0.71  cos(180°)=-1.0
  関係なし       やや似ている    完全に一致     やや逆方向           正反対
```

角度が小さいほど（同じ方向を向いているほど）コサイン類似度は 1 に近づき、
角度が大きくなるほど（逆方向に向かうほど）-1 に近づきます。

これをグラフで表すとこうなります：

```
cos(θ)
  1.0 ┤● θ=0°（完全に一致）
      │  ＼
  0.5 ┤    ＼
      │      ●  θ=60°
  0.0 ┤────────●────────── θ=90°（関係なし）
      │          ＼
 -0.5 ┤            ●  θ=120°
      │              ＼
 -1.0 ┤                ● θ=180°（正反対）
      └──┬─────┬─────┬─────┬─── θ
        0°    60°   90°  120°  180°
```

### なぜ「長さ」を無視するのか

「内積だけ使えばいいのでは？」と思うかもしれません。でも内積だと、ベクトルが長い（つまり文章が長い）ほど値が大きくなってしまいます。

コサイン類似度はベクトルの長さで割ることで**正規化**し、純粋に「向きの近さ」だけを測ります。文章の長さに関係なく、意味の近さだけで比較できます。

---

## もう一つのつまずき：類似度と距離が逆

「コサイン類似度が高い = 似ている」はわかった。でもコードを書くと「コサイン距離」というものも出てきます。最初、これで混乱しました。

| 指標 | 計算式 | 範囲 | 解釈 |
|------|--------|------|------|
| **コサイン類似度** | A·B / (\|A\|×\|B\|) | -1〜1 | **大きいほど似ている** |
| **コサイン距離** | 1 − コサイン類似度 | 0〜2 | **小さいほど似ている** |

```
コサイン類似度 = 0.95 → とても似ている（1に近い）
コサイン距離  = 1 - 0.95 = 0.05 → とても近い（0に近い）
```

「似ているほど値が大きい」のが類似度、「似ているほど値が小さい」のが距離です。最初に逆に覚えてしまい、クラスタリングの結果が全く意味をなさなくて困りました。

クラスタリングや最近傍探索のライブラリは「距離」を入力に取ることが多いので、コサイン類似度を計算した後に変換が必要なケースがよくあります。

```python
import numpy as np

similarity = np.dot(A, B) / (np.linalg.norm(A) * np.linalg.norm(B))
distance   = 1 - similarity  # ライブラリが距離を要求する場合はこちらを渡す
```

### 補足：コサイン距離は厳密には「距離」ではない

数学的な距離が満たすべき条件（三角不等式）を満たしていないため、厳密には「疑似距離」です。ただし実用上は問題なく使われています。

---

## 使い方

scikit-learn を使うと1行で計算できます。

```python
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

# 2つのベクトルの類似度
A = np.array([[1.0, 0.5, 0.2]])
B = np.array([[0.9, 0.6, 0.1]])
print(cosine_similarity(A, B))  # [[0.998...]]

# 複数ベクトル間の類似度行列
# embeddings: shape (N, D) の行列
sim_matrix = cosine_similarity(embeddings)  # shape (N, N)
```

`cosine_similarity` は行列同士の類似度を一括計算します。N件のデータがあると N×N の行列が返ってくるため、Nが大きいとメモリに注意が必要です。
